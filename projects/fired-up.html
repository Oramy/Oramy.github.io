Metrics

{% assign image_list = " /assets/images/fired_up/design_documents/camera_metrics_square.png, /assets/images/fired_up/design_documents/camera_ground_square.png, /assets/images/fired_up/design_documents/metrics_square.png " | split: "," %} {% assign caption_list = " Camera minimal zoom to ensure objects that are big enough on screen; Camera projection on ground to build well sized levels; Objects size (non-interactable in blue, interactable in yellow) " | split: ";" %} {% include image_carousel.html images=image_list captions=caption_list carousel-id="fired_up_carousel_metrics" %}
Camera

We defined camera's maximum range, object and character sizes to ensure game readability. This led to our level size as we wanted one-screen levels.

Level Metrics
Walls' height and thickness were then defined on the camera view to minimize blind spots while still ensuring circulation was clear.
We used a base unit (1m) to build our levels.

Helpers

I made helpers to ensure those metrics were respected when we integrated content and designed levels.
